# The Geometry of Relativity — code & reproducibility

Code, data artifacts, and the paper source for **"The Geometry of Relativity: Context-Relative
Scalar Representations in LLMs."** When given in-context examples that define a comparison class,
LLMs judge gradable adjectives (*tall/short*, *heavy/light*, …) by a context-normalized standing
`z = (x − μ)/σ` rather than the raw value `x`. This `z` is linearly decodable, causally steerable,
and partly shared across concepts; a small set of attention heads is causally responsible for it.

> **Status.** `make paper` builds the PDF with an audited bibliography; `make figures` regenerates
> the figures on CPU from committed results (no model needed); the prompt engine is unified and
> tested. About 24 of the paper's figures regenerate from data — three are committed assets (a
> hand-made pipeline schematic, one phase panel, and one shared-direction figure whose filename
> drifted from its generator). See `FIGURE_MANIFEST.md` for the per-figure map.

## Layout

```
config/      canonical model IDs and global seed
data_gen/    prompts.py — the unified prompt engine (legacy implicit + naturalistic frames)
figures/     build_paper_result_figures.py (behavioral) + build_circuit_figures.py (section 5)
circuit/     section-5 plotters (scripts/), their JSON/npz results, and per-model v13 inputs
extract/     GPU model-forward / intervention scripts (from-scratch tier)
internal/    phase-experiment data + helper the phase figure imports
results/     committed behavioral JSON/JSONL summaries
paper/       buildable LaTeX (main.tex, references.bib, icml2026 style, figures/)
tests/       prompt-engine golden tests
```

## Reproduce

```bash
python -m venv .venv && source .venv/bin/activate
pip install -e .            # numpy/scipy/sklearn/matplotlib/PIL — CPU analysis + plotting
```

1. **Figures (CPU, no model).** `make figures` regenerates the behavioral and circuit figures from
   the committed JSON/npz results into `paper/figures/` (a couple of minutes).
2. **Paper PDF.** `make paper` (tectonic) builds `paper/main.pdf` with the audited bibliography.
   **`make figures && make paper` reproduces the paper end to end.**
3. **From scratch (GPU).** The model forward passes + interventions that *produced* the cached
   results live in `extract/` and `circuit/scripts/` (the `*_extract_*` / `vextract` / `p2_extract_*`
   / `p2c_ablate_*` scripts). They need a GPU (an H100, or a 5090 for the 9B model). Cached results
   are committed so tiers 1–2 work without one; raw activation caches (Gemma-Scope SAE residuals,
   full attention dumps) are intentionally not committed.

   **Verified:** `make extract` regenerates the prompts + LD arrays on gemma-2-9b and reproduces the
   committed data (byte-identical prompts; LD corr 0.999 on a 5090). The full intervention pipeline
   and the behavioral tier are documented in `extract/PIPELINE.md`.

`pip install -e ".[gpu]"` adds torch/transformers for tier 3.

## Models

Primary: **`google/gemma-2-9b`**. Cross-model replication: Qwen2.5-3B, Llama-3.2-3B, OLMo-2-7B,
Pythia-2.8B, Qwen3-14B. Canonical IDs in `config/models.py`; the causal trio (L23H14/L26H4/L31H3)
and steering layer are there too.
